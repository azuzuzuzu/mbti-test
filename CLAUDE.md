# MBTI 测试网站项目

## 项目概述
这是一个 MBTI 性格测试网站，用户可以通过回答问题来测试自己的 MBTI 类型。

## 技术栈

### 后端
- **Go (Golang)** - 后端 API 服务器
  - 高性能、并发处理能力强
  - 编译型语言，部署简单（单个可执行文件）
- **Gin** - Go 的 Web 框架（推荐使用）
- **GORM** - Go 的 ORM 库，用于数据库操作
- **PostgreSQL** 或 **SQLite** - 数据库

### 前端
- **React 18+** - 用户界面库
- **TypeScript** - 类型安全的 JavaScript
- **Vite** - 现代化的前端构建工具（快速、轻量）
- **Tailwind CSS** - 实用优先的 CSS 框架
- **shadcn/ui** - 基于 Radix UI 的组件库
- **React Router** - 前端路由

## 开发指导原则

### 针对初学者的重要说明

1. **代码要简单明了**
   - 每个函数只做一件事
   - 变量命名要清晰（用完整的英文单词）
   - 添加必要的注释解释代码的作用

2. **循序渐进开发**
   - 先实现最基本的功能（MVP - 最小可行产品）
   - 测试通过后再添加新功能
   - 不要一次性写太多代码

3. **详细解释**
   - 在开发过程中，AI 助手会解释：
     - 每个文件的作用
     - 代码的工作原理
     - 为什么这样写
     - 如何运行和测试

4. **错误处理**
   - 遇到错误不要慌张
   - 仔细阅读错误信息
   - AI 助手会帮助解决问题

### 代码风格

**Go 代码：**
- 使用 `gofmt` 格式化代码
- 遵循 Go 官方命名规范
- 错误处理不能忽略

**React/TypeScript 代码：**
- 使用函数组件和 Hooks
- 组件名用 PascalCase（如 `TestPage`）
- 文件名用 kebab-case（如 `test-page.tsx`）
- 使用 Tailwind 类名而不是自定义 CSS

### 开发流程

1. **后端先行**：先开发 API 接口
2. **前端对接**：前端调用后端 API
3. **测试验证**：确保功能正常工作
4. **逐步完善**：添加样式和用户体验优化

## 部署计划

- **后端**：可部署到 Render、Railway、Fly.io（都有免费额度）
- **前端**：可部署到 Vercel、Netlify（免费）
- **数据库**：可使用 Supabase、PlanetScale（免费）

## AI 助手开发规则

当为这个项目开发代码时：

1. **必须提供详细解释**
   - 解释每个新概念
   - 说明代码的作用
   - 提供运行命令的说明

2. **代码要简洁**
   - 只写必要的代码
   - 避免过度设计
   - 不添加用户未要求的功能

3. **分步骤进行**
   - 一次只实现一个功能
   - 确保每一步都能运行
   - 提供测试方法

4. **使用中文解释**
   - 代码注释可以用中文
   - 所有说明用中文
   - 技术术语保留英文但要解释含义
